Claude Code Changelog

v2.1.212

Version 2.1.212 of Claude Code introduces several changes: the /fork command now copies conversations into a new background session (visible in 'claude agents') instead of launching an in-session subagent, with the old behavior moved to /subtask. It adds 'claude auto-mode reset' to restore default auto-mode configuration with a confirmation prompt (skippable with --yes). A session-wide limit on WebSearch tool calls was added (default 200, configurable via CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION) to prevent runaway search loops. A per-session cap on subagent spawning was also added, though the changelog text appears truncated.

Why it matters

This is a solid quality-of-life and safety update. The /fork change to background sessions is a meaningful workflow improvement, letting users continue working while a forked conversation runs independently. The web search and subagent caps are sensible guardrails against runaway resource consumption. The auto-mode reset command is a nice convenience feature. Overall, these are practical, well-thought-out improvements that address real usage concerns.
