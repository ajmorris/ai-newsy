Claude Code Changelog

v2.1.202

Version 2.1.202 of Claude Code adds a 'Dynamic workflow size' setting in /config to control agent counts in dynamic workflows (small/medium/large), adds OpenTelemetry attributes (workflow.run_id and workflow.name) for tracing workflow-spawned agent activity, and fixes a crash in inline Ctrl+R history search when accepting or cancelling during an ongoing scan.

Why it matters

This is a modest incremental release with useful improvements. The dynamic workflow size setting gives users more control over resource usage, the OpenTelemetry attributes are valuable for observability and debugging in production environments, and the Ctrl+R crash fix addresses a real usability issue. Nothing groundbreaking, but solid quality-of-life and reliability improvements.
