Claude Code Changelog

v2.1.214

Version 2.1.214 is a security-focused patch release for Claude Code that fixes several permission-check vulnerabilities: a bug where single-segment glob allow rules (like `src/`) would incorrectly auto-approve writes to nested directories anywhere in the file tree instead of only the intended root directory; a permission-check bypass specific to Windows PowerShell 5.1 sessions; a fix for Bash permission checks to fail closed on file-descriptor redirect forms where bash's parsing differs from the permission analyzer's parsing; and a fix for Bash permission checks misjudging very long commands, now requiring commands over 10,000 characters to always prompt for approval.

Why it matters

This is a critically important security patch release. The bugs fixed here are all related to permission-check bypasses, which are serious security concerns for a tool that can execute code and modify files. The glob pattern matching bug could have allowed unintended file modifications across the project tree, and the PowerShell and Bash permission check bypasses could have been exploited to run unauthorized commands.
